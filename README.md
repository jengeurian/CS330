1.	How do I approach designing software?
o	What new design skills has your work on the project helped you to craft?
o	What design process did you follow for your project work?
o	How could tactics from your design approach be applied in future work?
The approach to designing software is to maintain modularity to help improve the flow of the code while maintaining functionality, and ensuring that the same code doesn’t need to be written over and over again. During this assignment, I was given a good reminder to pay attention to my data types (float vs double) to prevent changing the data type in the middle of the code and avoid any issues that could come with that. Also, I learned that when building a 3D scene, it tends to work best to build the scene from the bottom to the top. This helps prevent the need to move multiple objects if you find an object isn’t in the placement you need it to be. As far as applying the design approach to future work, the bottom up approach can be used in so many ways. It is an excellent metaphor for coding in general. Before you start working on something that needs a stable base to support it, you should get the base in place first so you have the support you need later on. Whether this be in 3D design and placement of objects rendered in a scene, or in software design and writing the code for extra features before having the code in place for the basic software that will support that feature.


1.	How do I approach developing programs?
o	What new development strategies did you use while working on your 3D scene?
o	How did iteration factor into your development?
o	How has your approach to developing code evolved throughout the milestones, which led you to the project’s completion?
Working on this project really reinforced the idea of iteration for me. During the entire process, every time I wrote new code for a different component of the scene I would then run the code to see where the placement of the object was, how the texture was being applied, or whatever else I was changing. Then I would make the necessary changes within that code block to tweak the assignment as needed and run the code again. Constant testing of the code and only building one part at a time was hugely important to getting this project to come out anywhere near what I was aiming for.
